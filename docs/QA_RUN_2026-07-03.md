## Defects Found

### D-1 (P0, fixed): Dockerfile PHP version incompatible with composer.lock

- **Symptom:** `docker compose exec app composer install` failed — `composer.lock` (generated on host PHP 8.5.7) locks Symfony 8 / Laravel 13 packages requiring PHP ≥ 8.4.1, but `docker/php/Dockerfile` pinned `php:8.3-fpm`.
- **Why never caught:** the compose stack had never been executed (Docker Desktop was uninstalled until now); the file had only ever been YAML-syntax-checked.
- **Fix:** `docker/php/Dockerfile` base image bumped `php:8.3-fpm` → `php:8.5-fpm` (matches the host PHP the lockfile targets; `composer.json` requires `^8.3`, satisfied). Rebuilt; `composer install` then completed cleanly on PHP 8.5.7.
- **Scope:** infrastructure config only — no application code, schema, or feature change.

### D-2 (P0, fixed): DemoDataSeeder creates every demo user as `pending` — none can log in

- **Symptom:** `POST /api/login` for `supervisor@` / `employee@` / `intern@` / `marketer@` / `hr@timeforge.test` all returned "Your account is not active yet." Only the base-seeded admin (created via `User::factory()->admin()`) was active. Five of the six documented demo credentials (docs/DEMO.md, QA checklist header) were unusable — blocking every checklist phase past 0 and the entire demo script.
- **Root cause:** `DemoDataSeeder::ensureUser()` omitted `status` from the `firstOrCreate` attributes, so users inherited the schema default `pending`.
- **Why never caught:** the seeder had never been executed against a live database (no Docker, no host MySQL until this run), and no automated test covered it.
- **Fix:** `'status' => UserStatus::Active` added to `ensureUser()` in `backend/database/seeders/DemoDataSeeder.php`; regression test added (`tests/Feature/DemoDataSeederTest.php` — seeds both seeders, asserts all six users are active and each can log in via `POST /api/login`). Database reseeded; all six statuses verified `active` in MySQL and all six logins verified over HTTP. Full backend suite now **178/178** green.
- **Scope:** dev/demo-only seeder + one focused test — no production code path, schema, or feature change.
